  • @instw0 #8075 05:14 PM, 09 Oct 2024
    can I put a read/execute hook(!epthook) on a section of code (for example, the text ntoskrnl section)
  • @HughEverett ↶ Reply to #8075 #8076 05:36 PM, 09 Oct 2024
    Yes. That's what the '!monitor' command is designed for.

    https://docs.hyperdbg.org/commands/extension-commands/monitor
    !monitor (monitor read/write/execute to a range of memory) | HyperDbg Documentation

    Description of the '!monitor' command in HyperDbg.

  • @HughEverett ↶ Reply to #8075 #8077 05:37 PM, 09 Oct 2024
    But just keep in mind that putting a !monitor hook on a long range of memory is only possible if all of those pages are available at RAM (not paged-out).
  • @HughEverett ↶ Reply to #8075 #8079 05:40 PM, 09 Oct 2024
    So, if you put a !monitor hook on a range in which one or some of pages are not residing in RAM, then HyperDbg shows you error. In that case, you need to use the '.pagein' command to force Windows to bring those pages into the RAM.

    https://docs.hyperdbg.org/commands/meta-commands/.pagein
    .pagein (bring the page into the RAM) | HyperDbg Documentation

    Description of the '.pagein' command in HyperDbg.
